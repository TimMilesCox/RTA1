RTA1 fponline

What's fponline?

fponline is an algorithm application with a client on personal computers and a server on RTA1

fponline uses RTA1 192-bit floating arithmetic

fponline PC client orders input expressions of multiple terms and transmits the total request as a sequence of command steps to RTA1

fponline is like this to use 


	$ fponline -QR 172.29.7.25
	3.666666999999333333666666999999333333666666e1000000 / 3e-260000
	+1.222222333333111111222222333333111111222222e+1260000
	* 2e-260000 + 2e1000000
	+4.444444666666222222444444666666222222444445e+1000000
	for_later
	90 * 7 + 135
	+765.000
	_sine
	+7.071067811865475244008443621048490392848359e-1
	_square
	+0.500
	.
	$

	fponline -QR 172.29.7.7
	_root
	+7.071067811865475244008443621048490392848359e-1
	_cosine 60
	+5.000e-1
	_cube 3000000000000
	+2.700e+37
	_root3
	+3000000000000.000
	for_later / 2e-250000 + 5e1250000
	+7.222222333333111111222222333333111111222228e+1250000
	.
	$

fponline server in RTA1 responds with the resulting value

fponline client may capture and name resulting values for reinsertion and for reduction of complex expressions to simpler steps

Values may be supplied by name as expression tokens

Some trigonometric, geometric and volumetric commands may be applied to output values and to input expressions

Named values and a running total are saved between sessions for each individual user

The client platform keeps user data. The server only computes

fponline client and RTA1 emulated machine with fponline server may run on a single platform or on separate networked platforms

fponline client runs in a shell. It is run from directory rta/client because it references files in that directory. The two processes which comprise RTA1 machine must be running already in two other shell windows 


	cd	rta/portal	# server platform
	./runL			# RTA1 interface to physical networks

	_____________________________________________________________


	cd	rta/tgen	# server platform
	./rc			# RTA1 instruction processor and memory


	______________________________________________________________


	cd	rta/client	# client platform
	./prun			# routing to RTA1 wherever it is

	fponline

Where can I have it?

Like RTA1 emulated machine, fponline client requires POSIX services and currently is available on OSX and Linux. RTA1 download contains simple guidelines for installing openSUSE Linux on PCs which do not have OSX

RTA1 emulator and fponline client are available for PowerPC and x86 OSX and for x86 Linux. Some 64-bit Linux distributions need a runtime download for 32-bit RTA binaries

Some more Things fponline does

How does it work?

The PC just-in-time assembles input expressions using RTA1 development tools. The assembly output is a protocol data unit transmitted to RTA1

The PDU is assembled onto a RAM file system to avoid harsh demands on the PC's disk or NVRAM file system. There is at most one assembly per input, but scripts can be piped into fponline client, resulting in multiple assembly outputs per second

The user's named values and running total are on files which do not write through until end of session. This is achieved with POSIX feature file memory mapping

The just-in-time assembly uses algorithm planning features in RTA1 development tools to turn the expression into ordered steps, and scans number strings and names into RTA1-format 192-bit floating point. fponline server on RTA1 is a UDP request / reponse paradigm which computes the requested steps in order and returns the result as edited text. At this point the user may attach a name to the output

A Helpful Hint or Two

fponline is script-capable. If scripts are repeated mutiple times and save many names, performance eventually slows unless duplicate names are removed from the user's list

Just-in-time assembly equates duplicate names as many times as they are duplicated. It may be an advantage to remove some older definitions, because 192-bit floating point equates are not trivial for the PC and thousands of them can take visible moments 


	edit	~/dynamic.rta/my_names.def	# any text editor you like of course
You can even delete your list of saved names when it gets large. But back it off first because you might forget something 

	cp -r	~/dynamic.rta	somewhere.else
	rm	~/dynamic.rta/my_names.def
You can also force a complete initialisation of your fponline handwork by removing directory dynamic.rta completely. It configures itself again with your running total reset to 0.0 and no saved names. Again, save it first in case you forget anything 

	cp -r	~/dynamic.rta	somewhere.safe
	rm -Rf	~/dynamic.rta
