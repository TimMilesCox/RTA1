

        added on-the-fly-spy to instruction inputs and results
        different from breakpoint and step-by-step 
        on-the-fly-spy outputs while running

        compile it into the emulator with

                rta/rta.run$ tipgenro -DEDGE

        turn it on with RTA target code instruction

                on      -32768

        turn it off with RTA target code instruction

                off     -32768

        so far only displays instructions dl da dan qpop
        for example

                on      -32768
                dan     subtrahend
                off     -32768

        because it shows all the inputs and outputs
        to targeted instructions, it has the effect
        of tracing the previous instruction result 
        as well as the current present instruction
