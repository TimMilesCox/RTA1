1. RTA1 Embedded Target Software
________________________________

complete datagram reassembly and source fragmentation
	devise a way of testing
	define and devise restriction on accepting fragments

add ip MIB to MIB-II implementation, at least ipAddr and maybe ipRoute
populate more of MIB-II statistical counters
	see if netSNMP client manager can cope with number results up to 6 octets

complete Very High Volume TCP/IP buffering and devise tests


very possibly explore ZeroConf for emulated RTA1
this may be defeated by the fact that emulated machine's net addresses
are necessarily alias addresses of the emulator host's interfaces

both emulated machine and emulator host are currently capable of
answering ARP for any address used by the emulated machine.

The idea falls down if ZeroConf has any directed ARP datagrams in it
because the Zero Configuration traffic needs to be shown to the
emulated machine (ARP broadcasts are), without the emulator host
configuring the ZeroConf address to itself

It is also a conceivable idea, just to give the whole duty of
ARP back to the emulator host, since it owns the addresses used
by RTA1

or alternatively, never to configure RTA1's protocol addresses to
the emulator host, which then still needs to drop RTA1's traffic
from the kernel via firewall, to avoid ICMP DUs and TCP_RSTs
going out from the emulator host

2. Emulated RTA1 Network Interfacing for Ubuntu and Windows 10
_______________________________________________________________

attempt to have complete emulation available on Linux and Windows 10
by means of BPF based network interfacer portal/osserv

historically only a BPF based network interface has worked on every
Apple OSX release, since IPFW is withdrawn and and PCAP never worked
completely on some OSX distributions

Ubunto is expected to have BPF. It's hoped a complete BPF API can be
found in the independent software Windows 10 PCAP

Overall purpose is more exactly to have softcore and eventually fabbed
RTA1 rather than many emulations, but an opportunity is perceived in
having RTA emulated on cheap computers running Windows 10, because of
the crisp performance of code written in C, compiled with VC++ cl compiler
and running in command prompt

RTA1 might then be shown to many more engineers
